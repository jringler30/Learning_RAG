DOCUMENT: Introduction to Machine Learning
Machine learning is a subset of artificial intelligence that enables computers to learn from data without being explicitly programmed. Instead of writing rules manually, machine learning algorithms identify patterns in training data and use those patterns to make predictions on new, unseen data. The three main types of machine learning are supervised learning, unsupervised learning, and reinforcement learning. Supervised learning uses labeled data where the correct answers are known. Unsupervised learning finds hidden structure in unlabeled data. Reinforcement learning trains agents to make decisions by rewarding good behavior and penalizing bad behavior.

DOCUMENT: Neural Networks and Deep Learning
Neural networks are computational models inspired by the structure of the human brain. They consist of layers of interconnected nodes called neurons. Deep learning refers to neural networks with many layers, which allows them to learn hierarchical representations of data. The input layer receives raw data, hidden layers transform the data through learned weights and activation functions, and the output layer produces predictions. Backpropagation is the algorithm used to train neural networks by computing gradients and updating weights to minimize a loss function. Common activation functions include ReLU, sigmoid, and tanh.

DOCUMENT: Transformer Architecture
The Transformer architecture, introduced in the paper "Attention Is All You Need" (2017), revolutionized natural language processing. Unlike recurrent networks, Transformers process entire sequences in parallel using a mechanism called self-attention. Self-attention allows each token in a sequence to attend to every other token, capturing long-range dependencies efficiently. The architecture consists of an encoder and decoder, each containing multi-head attention layers and feed-forward networks. Most modern large language models like GPT and BERT are based on the Transformer architecture.

DOCUMENT: Large Language Models
Large language models (LLMs) are neural networks trained on massive text datasets with billions of parameters. They are pre-trained using self-supervised objectives like next-token prediction, then fine-tuned for specific tasks. Examples include GPT-4, Claude, Llama, and Gemini. LLMs exhibit emergent capabilities — abilities that were not explicitly trained for but arise from scale. Key capabilities include question answering, summarization, code generation, translation, and reasoning. Prompt engineering is the practice of crafting inputs to guide LLM behavior. In-context learning allows LLMs to adapt to new tasks from a few examples in the prompt without weight updates.

DOCUMENT: Retrieval-Augmented Generation
Retrieval-Augmented Generation (RAG) is a technique that combines a retrieval system with a generative language model. When a user asks a question, RAG first searches a knowledge base for relevant documents, then passes those documents as context to the LLM when generating a response. This approach grounds the model's answers in factual documents, reducing hallucinations and enabling the model to answer questions about information not in its training data. RAG was introduced by Facebook AI Research in 2020. The two main components are the retriever (often a dense vector search) and the generator (an LLM).

DOCUMENT: Vector Embeddings
Vector embeddings are numerical representations of data (text, images, audio) in a high-dimensional vector space. Similar items are mapped to nearby points in this space. Text embeddings are created by passing text through an encoder model. The resulting vectors capture semantic meaning — so "king" and "queen" are closer together than "king" and "banana". Embeddings enable semantic search, where you can find relevant documents based on meaning rather than exact keyword matches. Popular embedding models include OpenAI's text-embedding-3, Sentence-BERT, and E5. Embedding dimensions typically range from 384 to 3072.

DOCUMENT: Vector Databases
Vector databases are storage systems optimized for storing and searching high-dimensional vectors. Unlike traditional databases that use exact match lookups, vector databases use approximate nearest neighbor (ANN) algorithms to find the most similar vectors to a query. Common algorithms include HNSW (Hierarchical Navigable Small World) and IVF (Inverted File Index). Popular vector databases include Pinecone, Weaviate, Qdrant, Milvus, and ChromaDB. They support metadata filtering, allowing you to combine semantic search with attribute-based filtering. Vector databases are a core component of RAG systems.

DOCUMENT: Chunking Strategies for RAG
Chunking is the process of splitting documents into smaller pieces for storage and retrieval in a RAG system. The chunk size affects retrieval quality significantly. Small chunks are more precise but may lack context. Large chunks provide more context but may be less precise and exceed LLM context windows. Common strategies include fixed-size chunking (splitting by character or token count), sentence-based chunking (splitting at sentence boundaries), and recursive chunking (splitting by paragraph, then sentence, then word). Overlap between chunks (e.g., 20% overlap) helps preserve context at chunk boundaries.

DOCUMENT: Evaluating RAG Systems
RAG systems are evaluated on retrieval quality and generation quality. Retrieval metrics include recall (did we retrieve the relevant documents?) and precision (were the retrieved documents relevant?). Generation metrics include faithfulness (is the answer grounded in the retrieved documents?), answer relevance (does the answer address the question?), and context relevance (are the retrieved documents relevant to the question?). The RAGAS framework provides automated metrics for evaluating RAG pipelines. Human evaluation remains important for assessing coherence and usefulness.

DOCUMENT: Fine-Tuning vs RAG
Fine-tuning and RAG are two complementary approaches for adapting LLMs to specific domains. Fine-tuning updates the model's weights on domain-specific data, teaching the model new knowledge and styles. RAG keeps the model weights frozen and retrieves relevant context at inference time. Fine-tuning is better for adapting writing style, format, and task-specific behavior. RAG is better for keeping information up-to-date and providing access to large, frequently changing knowledge bases. In practice, many production systems combine both: a fine-tuned model with a RAG retrieval system.
